# RS3 Achievements Tracker — Backend

Spring Boot REST API for tracking RuneScape 3 player achievements, archaeology logs, and hiscores.

## Tech Stack

- **Java 21** + **Spring Boot 3.3.4**
- **Spring Security** with JWT authentication
- **PostgreSQL** + **Flyway** migrations
- **Maven**
- **Docker** support

## Getting Started

### Prerequisites

- Java 21
- Maven 3.9+
- PostgreSQL 14+

### Local Setup

1. Create a PostgreSQL database:
   ```sql
   CREATE DATABASE fullstackapp;
   ```

2. Run the app (uses sensible local defaults):
   ```bash
   mvn spring-boot:run
   ```

   The API starts on `http://localhost:8080`.

### Environment Variables

Override these for production (all required when using the `prod` profile):

| Variable      | Description                        |
|---------------|------------------------------------|
| `DB_HOST`     | Postgres host                      |
| `DB_PORT`     | Postgres port                      |
| `DB_NAME`     | Database name                      |
| `DB_USERNAME` | Database user                      |
| `DB_PASSWORD` | Database password                  |
| `JWT_SECRET`  | Base64-encoded 256-bit JWT secret  |

> **Security note:** Never commit real credentials. Always supply secrets via environment variables in production.

### Docker

```bash
docker build -t rs3-tracker-backend .
docker run -p 8080:8080 \
  -e DB_HOST=host.docker.internal \
  -e DB_NAME=fullstackapp \
  -e DB_USERNAME=postgres \
  -e DB_PASSWORD=yourpassword \
  -e JWT_SECRET=yoursecret \
  rs3-tracker-backend
```

## Running Tests

```bash
mvn test
```

Code coverage reports are generated by JaCoCo in `target/site/jacoco/`.

## API Overview

| Method | Endpoint                     | Description                     |
|--------|------------------------------|---------------------------------|
| GET    | `/health`                    | Health check                    |
| POST   | `/api/auth/register`         | Register a new account          |
| POST   | `/api/auth/login`            | Login, returns JWT              |
| GET    | `/api/me`                    | Current user info (auth)        |
| POST   | `/api/claim`                 | Claim a RS3 character (auth)    |
| GET    | `/api/player/{name}`         | Player profile & hiscores       |
| GET    | `/api/archaeology`           | Archaeology log entries (auth)  |
| POST   | `/api/archaeology`           | Add archaeology entry (auth)    |
| DELETE | `/api/archaeology`           | Remove archaeology entry (auth) |

## Database Migrations

Flyway migrations live in `src/main/resources/db/migration/`. They run automatically on startup.
